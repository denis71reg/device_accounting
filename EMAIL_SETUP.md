# Настройка Email-уведомлений

Приложение отправляет email-уведомления супер-админу при удалении любых объектов в системе.

**Письма отправляются с адреса `da@ittest-team.ru` на email супер-админа из базы данных.**

## Настройка SMTP

Настройки уже настроены для работы с корпоративной почтой `da@ittest-team.ru`.

Добавьте в файл `.env` на сервере только пароль (остальные настройки уже установлены по умолчанию):

```bash
# SMTP настройки
SMTP_PASSWORD=your-smtp-password-here
```

Или если нужно переопределить настройки:

```bash
SMTP_HOST=mail.ittest-team.ru
SMTP_PORT=465
SMTP_USER=da@ittest-team.ru
SMTP_PASSWORD=your-smtp-password-here
```

**Важно:** Пароль содержит специальные символы, убедитесь, что он правильно экранирован в `.env` файле.

## Тестирование отправки email

Для проверки работы email уведомлений можно использовать тестовый скрипт:

```bash
python test_email.py
```

Скрипт отправит тестовое уведомление на email супер-админа из базы данных.

## Проверка работы

После настройки при удалении любого объекта (девайс, сотрудник, склад и т.д.) на указанный email придет уведомление с информацией:
- Тип удаленного объекта
- Название объекта
- Кто удалил
- Тип удаления (мягкое или окончательное)

## Отключение уведомлений

Если не указать `SMTP_HOST`, `SMTP_USER` или `SMTP_PASSWORD`, уведомления отправляться не будут, но приложение продолжит работать нормально. Ошибки отправки логируются, но не прерывают работу приложения.
