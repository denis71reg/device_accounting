## Что нужно сделать СРОЧНО

### Вариант 1: Сделать репозиторий приватным (РЕКОМЕНДУЕТСЯ)
1. Перейдите на https://github.com/denis71reg/device_accounting
2. Settings → General → Danger Zone → Change repository visibility
3. Выберите "Make private"

### Вариант 2: Удалить пароли из истории Git (сложнее)
Если репозиторий должен остаться публичным, нужно:
1. Изменить все пароли (SSH и SMTP)
2. Использовать `git filter-branch` или `BFG Repo-Cleaner` для удаления паролей из истории
3. Сделать force push (это перезапишет историю)

**⚠️ ВАЖНО:** После удаления паролей из истории нужно изменить их на сервере!

## Рекомендации на будущее

1. **Никогда не коммитьте пароли в Git**
2. Используйте `.env` файлы (они в `.gitignore`)
3. Используйте переменные окружения на сервере
4. Используйте секреты в CI/CD системах (GitHub Secrets, GitLab CI Variables)
5. Для SSH используйте ключи вместо паролей

2. **Измените SMTP пароль:**

## Безопасное хранение паролей

- Используйте менеджеры паролей (1Password, LastPass, Bitwarden)
- Используйте секреты в CI/CD
- Используйте переменные окружения на сервере
- Используйте SSH ключи вместо паролей
