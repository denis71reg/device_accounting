# Деплой сейчас

## Проблема
SSH порт 22 недоступен с вашей машины (таймаут). Возможные причины:
- Требуется VPN подключение
- SSH работает на другом порту
- Файрвол блокирует подключение

## Решение: Выполните деплой вручную

### Шаг 1: Подключитесь к серверу
```bash
ssh ittest@192.168.16.44
# Пароль: [удален из репозитория - используйте безопасное хранилище]
```

### Шаг 2: Загрузите файлы на сервер

**Вариант A: Через SCP (с другой машины, где есть доступ)**
```bash
scp deploy_package.tar.gz ittest@192.168.16.44:/opt/device_accounting/
```

**Вариант B: Через веб-интерфейс или другой способ**
- Загрузите `deploy_package.tar.gz` на сервер любым доступным способом

### Шаг 3: На сервере выполните
```bash
cd /opt/device_accounting
tar -xzf deploy_package.tar.gz
chmod +x deploy_remote.sh
./deploy_remote.sh
```

## Или используйте rsync (если SSH доступен с другой машины)
```bash
rsync -avz --progress \
    --exclude '.git' --exclude 'venv' --exclude '__pycache__' \
    --exclude '*.pyc' --exclude 'instance' --exclude '.env' \
    --exclude '*.db' --exclude '*.log' \
    ./ ittest@192.168.16.44:/opt/device_accounting/
```
